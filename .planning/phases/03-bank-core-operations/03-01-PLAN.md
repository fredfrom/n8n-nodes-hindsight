- Operation value: `'createOrUpdate'`
- Required field: `bankId` (string, required, `default: ''`, description: `'ID of the bank to create or update'`)

  - `mission` (string, `default: ''`, description: `'Mission statement for the bank'`)
  - `name` (string, `default: ''`, description: `'Display name for the bank'`)
  - Items MUST be alphabetized by `name` (mission before name) per Pitfall 15
  - Items inside collection MUST NOT have `required: true` per Pitfall 7
- API: `PUT /v1/default/banks/{bank_id}` with body `{ name?, mission? }`
- Do NOT expose deprecated fields (disposition, background, disposition_skepticism, disposition_literalism, disposition_empathy) per Pitfall 5

**list.ts** (per BANK-02):
- Operation value: `'list'`
- Export an empty array `[]` — List Banks has no parameters (the API takes no query params)

**getProfile.ts** (per BANK-03):
- Operation value: `'getProfile'`
- Required field: `bankId` (string, required, `default: ''`, description: `'ID of the bank to get the profile for'`)

**getStats.ts** (per BANK-04):
- Operation value: `'getStats'`
- Required field: `bankId` (string, required, `default: ''`, description: `'ID of the bank to get statistics for'`)

**delete.ts** (per BANK-05):
- Operation value: `'delete'`
- Required field: `bankId` (string, required, `default: ''`, description: `'ID of the bank to delete'`)

**Lint rules to follow for ALL files:**
- `displayName` in Title Case; "ID" always uppercase (not "Id")
- Single-sentence descriptions: NO trailing period
- Boolean descriptions start with "Whether"
- `noDataExpression: true` on operation/resource dropdowns only (not on string fields)
- Use tabs for indentation (project convention from Phase 1)
